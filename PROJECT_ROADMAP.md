# CodeEdu 项目路线图

## 📊 当前完成状态总结

### ✅ 已完成的核心功能

#### 1. 后端架构与数据库
- **数据库设计**：10个完整的数据表（用户、题目、比赛、班级、提交等）
- **数据模型**：SQLAlchemy ORM 模型定义
- **数据库初始化**：包含测试数据的 SQLite 数据库 (`codeedu.db`)
- **API 服务器**：Flask 后端运行在 `http://localhost:5000`

#### 2. 用户系统
- **多角色权限**：学生(student)、教师(teacher)、管理员(admin)
- **认证机制**：JWT 令牌认证
- **用户管理**：8个测试用户（3个角色）
- **密码安全**：SHA-256 哈希存储

#### 3. 题目管理系统
- **题目CRUD**：完整的创建、读取、更新、删除操作
- **难度分类**：简单(easy)、中等(medium)、困难(hard)
- **标签系统**：题目分类和标签
- **题目数量**：16个编程题目（从6个扩展到16个）
- **测试用例**：每个题目包含多个测试用例

#### 4. 评测系统
- **评测API**：`POST /api/judge/run` 独立评测端点
- **详细结果**：每个测试用例的输入、预期输出、实际输出、状态
- **评测状态**：AC、WA、TLE、MLE、RE、CE 等
- **分数计算**：基于测试用例通过率自动计算分数
- **运行统计**：运行时间和内存使用统计

#### 5. 比赛管理系统
- **比赛创建**：支持时间控制（开始时间、结束时间）
- **题目关联**：比赛与题目多对多关系
- **参与管理**：用户报名参加比赛
- **排名系统**：基于分数和提交时间的自动排名
- **示例比赛**：2个完整的测试比赛

#### 6. 班级管理系统
- **班级创建**：教师创建和管理班级
- **学生注册**：学生加入班级
- **成绩统计**：班级内学生成绩统计
- **教师管理**：教师管理班级和学生
- **示例班级**：2个测试班级

#### 7. 数据管理工具
- **数据导入**：CSV文件导入用户、题目、班级
- **数据导出**：JSON格式完整数据导出
- **批量操作**：支持批量创建和更新
- **数据验证**：导入时的数据验证和去重

#### 8. 前端基础架构
- **API配置**：完整的TypeScript API接口定义
- **状态管理**：Pinia stores 用于用户、题目、比赛、班级
- **组件结构**：Vue 3 组件基础架构
- **路由配置**：Vue Router 基本配置

#### 9. 开发工具和文档
- **系统测试**：`test_system.py` 完整API测试
- **数据库检查**：`check_db.py`、`check_users.py` 数据库验证
- **题目添加**：`add_more_problems.py` 批量添加题目
- **项目文档**：4份完整技术文档
- **GitHub推送**：代码已成功推送到GitHub仓库

## 🔧 技术决策点

### 需要决策的技术问题
1. **前端框架选择**：继续使用Vue 3还是考虑React？
2. **UI组件库**：使用Element Plus、Ant Design Vue还是自研？
3. **代码编辑器**：Monaco Editor vs CodeMirror vs Ace Editor
4. **评测沙箱**：Docker vs Kubernetes vs 自定义沙箱
5. **数据库迁移**：何时从SQLite迁移到PostgreSQL/MySQL？

### 建议方案
1. **前端框架**：继续Vue 3（已有基础，团队熟悉）
2. **UI组件库**：Element Plus（功能丰富，中文文档好）
3. **代码编辑器**：Monaco Editor（VS Code同款，功能强大）
4. **评测沙箱**：Docker（成熟稳定，资源隔离好）
5. **数据库迁移**：用户量超过1000时考虑迁移

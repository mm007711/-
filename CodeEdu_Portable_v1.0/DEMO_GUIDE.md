# CodeEdu 演示指南 - 给院长的15分钟展示

## 🎯 演示目标

向院长展示 CodeEdu 在线评测系统的核心功能、技术架构和项目潜力，争取支持和资源。

## 📅 演示时间安排

**总时长：15分钟**

| 时间段 | 内容 | 重点 |
|--------|------|------|
| 0-2分钟 | 系统概览 | 项目定位、核心数据统计 |
| 2-5分钟 | 多角色演示 | 学生/教师/管理员不同权限 |
| 5-8分钟 | 代码评测演示 | 智能评测、详细结果反馈 |
| 8-10分钟 | 功能亮点 | 数据落库、评测隔离、班级管理 |
| 10-12分钟 | 技术架构 | 前后端技术栈、扩展性 |
| 12-15分钟 | Q&A环节 | 回答院长关心的问题 |

## 🎬 演示脚本

### 开场白 (30秒)
"院长好，今天向您展示我们开发的 CodeEdu 在线评测系统。这是一个完整的编程教学平台，支持算法竞赛、班级管理和智能评测。"

### 1. 系统概览 (1.5分钟)
**操作**：打开 `demo.html`，展示系统统计数据
**讲解要点**：
- 系统定位：智能在线编程评测与教学平台
- 核心数据：16个题目、8个用户、2个比赛、2个班级
- 目标用户：计算机专业学生、编程竞赛选手、教师

### 2. 多角色演示 (3分钟)
**操作**：切换不同角色并模拟登录
**讲解要点**：

**学生角色**：
- 权限：查看题目、提交代码、参加比赛、查看成绩
- 应用场景：日常练习、参加竞赛、查看学习进度

**教师角色**：
- 权限：创建题目、管理比赛、创建班级、批改作业
- 应用场景：布置作业、组织比赛、跟踪学生学习情况

**管理员角色**：
- 权限：系统管理、用户管理、数据统计、权限分配
- 应用场景：系统维护、数据分析、用户支持

### 3. 代码评测演示 (3分钟)
**操作**：运行三种不同的评测场景
**讲解要点**：

**正确代码 (AC)**：
- 展示详细测试用例结果
- 强调算法分析（时间复杂度O(n)，空间复杂度O(n)）
- 技术说明：Docker沙箱环境，安全隔离

**错误代码 (WA)**：
- 展示错误定位和反馈
- 强调系统的诊断能力
- 技术说明：多种比较模式，智能错误提示

**超时代码 (TLE)**：
- 展示时间限制和资源监控
- 强调性能优化指导
- 技术说明：实时监控，自动终止超时进程

### 4. 功能亮点 (2分钟)
**讲解要点**：

1. **完整数据落库**
   - SQLite数据库 + ORM模型
   - 10个数据表，完整的关系设计
   - 支持数据导入导出

2. **题目评测隔离**
   - 独立评测API
   - 沙箱环境，支持多语言
   - 安全可靠的代码执行

3. **详细判题结果**
   - 每个测试用例独立反馈
   - 输入、输出、预期、运行时间
   - 算法分析和优化建议

4. **竞赛管理系统**
   - 时间控制、题目关联
   - 自动排名、成绩统计
   - 实时更新排名榜

5. **班级管理**
   - 教师创建班级，学生注册
   - 成绩跟踪，学习分析
   - 作业批改和反馈

6. **响应式设计**
   - 支持PC、平板、手机
   - 现代化UI设计
   - 良好的用户体验

### 5. 技术架构 (2分钟)
**讲解要点**：

**后端技术栈**：
- Flask + SQLAlchemy ORM
- SQLite数据库（可扩展至PostgreSQL）
- JWT身份认证，RESTful API设计

**前端技术栈**：
- Vue.js 3 + TypeScript
- Pinia状态管理，Vite构建工具
- 响应式UI设计

**开发工具**：
- 数据导入导出工具
- 系统测试脚本
- 数据库管理工具
- 完整文档体系

**扩展性**：
- 模块化设计，易于扩展
- 支持微服务架构迁移
- 可部署到云平台

### 6. Q&A环节 (3分钟)
**预期问题及回答准备**：

**Q1：这个系统与现有商业平台（如LeetCode、牛客网）有什么区别？**
A：我们更注重教学场景，支持班级管理、作业批改、学习分析等教育功能，同时保持开源和可定制性。

**Q2：系统的安全性如何保障？**
A：采用JWT认证、代码沙箱执行、输入验证、SQL防注入等多层安全措施。

**Q3：系统能支持多少并发用户？**
A：当前架构支持数百并发，通过优化和扩展可支持数千并发。

**Q4：后续发展计划是什么？**
A：短期完善前端界面，中期支持更多编程语言，长期实现智能推荐和个性化学习路径。

**Q5：需要多少资源投入？**
A：当前已完成核心开发，需要1-2名开发人员完善前端，1名UI设计师优化界面。

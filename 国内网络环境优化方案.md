# CodeEdu 国内网络环境优化方案

## 问题分析

### 1. 国外依赖下载慢
- **npm包**：Vue、TypeScript等依赖从npm官方仓库下载慢
- **Python包**：Flask、pandas等从PyPI下载慢
- **GitHub**：项目模板和示例代码访问不稳定

### 2. 离线部署困难
- 依赖网络连接安装依赖
- 缺少完整的离线安装包
- 部署过程复杂

### 3. 用户体验问题
- 启动过程需要网络连接
- 错误提示不友好
- 缺少中文文档和界面

## 解决方案

### 一、依赖管理优化

#### 1.1 使用国内镜像源
**npm配置（.npmrc文件）**
```
registry=https://registry.npmmirror.com/
disturl=https://npmmirror.com/mirrors/node
electron_mirror=https://npmmirror.com/mirrors/electron/
electron_builder_binaries_mirror=https://npmmirror.com/mirrors/electron-builder-binaries/
```

**pip配置（pip.ini文件）**
```
[global]
index-url = https://pypi.tuna.tsinghua.edu.cn/simple
trusted-host = pypi.tuna.tsinghua.edu.cn
```

#### 1.2 创建依赖包缓存
- 将npm包和Python包打包到项目中
- 提供离线安装脚本
- 支持无网络环境部署

### 二、项目架构优化

#### 2.1 简化部署结构
```
CodeEdu_便携版/
├── 启动教学平台.bat          # Windows启动脚本
├── 启动教学平台.sh           # Linux启动脚本

├── backend/                 # 后端（包含所有依赖）

```

#### 2.2 预置依赖包
- 将Python虚拟环境打包
- 将node_modules打包
- 提供一键恢复脚本

### 三、用户体验优化

#### 3.1 中文界面支持
- 数据导入工具完全中文化
- 错误提示中文化
- 操作指南中文化

#### 3.2 离线功能增强
- 离线数据导入/导出

#### 3.3 简化启动流程
- 一键启动脚本

## 实施步骤

### 第一阶段：依赖镜像配置

#### 1. 创建npm配置文件
```bash
# 创建.npmrc文件
echo "registry=https://registry.npmmirror.com/" > .npmrc
echo "disturl=https://npmmirror.com/mirrors/node" >> .npmrc
```

#### 2. 创建pip配置文件
```bash
# Windows
echo [global] > %APPDATA%\pip\pip.ini
echo index-url = https://pypi.tuna.tsinghua.edu.cn/simple >> %APPDATA%\pip\pip.ini
echo trusted-host = pypi.tuna.tsinghua.edu.cn >> %APPDATA%\pip\pip.ini
```

### 第二阶段：离线包准备

#### 1. 下载npm包
```bash
# 在能访问外网的机器上
cd frontend
npm cache clean --force
npm install --registry=https://registry.npmmirror.com/
tar -czf node_modules.tar.gz node_modules/
```

#### 2. 下载Python包
```bash
# 创建requirements.txt
pip freeze > requirements.txt

# 下载所有包到本地
pip download -r requirements.txt -d ./python_packages --index-url https://pypi.tuna.tsinghua.edu.cn/simple
```

## 技术实现细节

### 1. 虚拟环境打包
```bash
# 创建便携式Python环境
python -m venv portable_venv
# 激活并安装依赖
portable_venv\Scripts\activate
pip install -r requirements.txt --index-url https://pypi.tuna.tsinghua.edu.cn/simple
# 打包虚拟环境
tar -czf portable_venv.tar.gz portable_venv/
```

### 2. Node.js依赖打包
```bash
# 使用npm pack打包所有依赖
cd frontend
npm pack
# 创建离线安装脚本
echo "tar -xzf node_modules.tar.gz" > install.sh
echo "npm rebuild" >> install.sh
```

### 3. 数据库预置
- 包含完整的测试数据
- 包含示例用户、题目、班级
- 包含管理员账户

### 4. 启动脚本优化
```batch
@echo off
chcp 65001 >nul
echo.
echo ========================================
echo    CodeEdu 教学平台 - 便携版
echo ========================================
echo.

REM 检查Python
if not exist "backend\portable_venv" (
    echo [错误] 未找到Python虚拟环境
    echo 请确保已正确解压所有文件
    pause
    exit /b 1
)

REM 启动后端
echo 正在启动后端服务器...
start "CodeEdu后端" cmd /k "backend\portable_venv\Scripts\python backend\simple_server.py"

REM 等待后端启动
timeout /t 3 /nobreak >nul

REM 启动前端
echo 正在启动前端界面...
start "CodeEdu前端" cmd /k "cd frontend && npm run dev"

echo.
echo 系统启动完成！
echo 后端：http://localhost:5000
echo 前端：http://localhost:5173
echo.
pause
```

## 故障排除

### 常见问题1：端口占用
```
解决方案：
1. 修改backend/simple_server.py中的端口号
2. 修改frontend/vite.config.ts中的代理配置
3. 重启服务
```

### 常见问题2：数据库锁死
```
解决方案：
1. 删除codeedu.db-journal文件
2. 从备份恢复数据库
3. 运行数据库修复工具
```

### 常见问题3：内存不足
```
解决方案：
1. 增加虚拟内存
2. 优化数据库查询
3. 减少并发用户数
```
